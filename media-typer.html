Media Typer

Media typer kan sees på som forløperen til det vi kaller media queries for responsivt design. Før media queries var det veldig vanlig å lage separate css stilark for alle de forskjellige enhetene, vi fikk derfor media typer som screen (for skjermer), handheld (for mobil) men og en hel del andre som for skjermlesere, braille for blindeskrift, og andre for spesielle bruksområder. Mange av disse er innebygget i den nyeste versjonen av CSS spesifikasjonen for media queries og kan taes i bruk direkte derifra.

Mange av disse er så å si ubrukt i dag, med mindre man har et spesifiukt bruksområde for dem. i dag bruker mest “all” for alle enheter, “screen” for skjermer og “print” for utskrift. Print varianten er fortsatt i bruk, ettersom den kan benyttes til å lage spesifikke design for utskrift. som f.eks om noen vill skrive ut en artikkel, så kan de lettere så en mer ryddig tekst om utvikleren har laget istand et oppsett. slik at arket brukerens utskrift ikke har massu unyttig design som tar opp plass.
